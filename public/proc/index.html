Proc

No todo en Ruby es un objeto, por ejemplo los bloques (blocks) de código. Sin embargo y por suerte pueden ser convertidos en objetos. ¿Cómo convertimos un bloque en un objeto? Tenemos 4 opciones: - Usando Proc.new

>> p = Proc.new { "hola" }
=> #<Proc:0xb71921b8@(irb):1>
>> puts p.call
hola
=> nil
  • Pasando el bloque como argumento de un método
def meth1(p1, p2, &block)
  # el & indica que ese argumento debe ser un bloque
  # y se debe convertir en un Proc
  puts block.inspect
end
meth1(1,2) { "a block" } #<Proc:0x0a4f4c@/tmp/prog.rb:4>
meth1(3,4) # nil

Esta forma es exactamente lo mismo que la primera. - Usando el método Kernel.lambda y asociandolo a un bloque.

>> block = lambda { "hola" }
=> #<Proc:0xb7334dcc@(irb):1>
  • Usando el método Kernel.proc (solo válido en 1.9)
def return_a_block(&block)
  block
end
def store_a_block(&block)
  @local = block
end
**Ejecutando el código de un Proc:** - yield 
def meth
  res = yield
  "The block returns #{res}"
end
- call 
l = Proc.new { |a| a + 1 }
puts l.call(99)
**Diferencias entre Proc.new & lambda** - Manejo de parámetros: Proc.new es muy liberal con la cantidad de parámetros (como paralell asigment). 
l = Proc.new { |a,b,c| puts a,b,c }
l.call(1,2,3)
l.call(1,2)
l.call(1,2,3,4)
lambda es más rígido (como la llamada a un método). 
l = lambda { |a,b,c| puts a,b,c }
l.call(1,2,3,4) -> error
l = lambda { |a,*b| puts a, b }
- Retorno (return) Proc.new toma el return y sale del contexto que lo rodea, mientras que en lambda, el return solo sale del Proc.: 
def method
  puts "at top"
  pr = Proc.new { return }
  puts "at the bottom"
end
puts "before call"
method
puts "after call"
# Salida:
before call - at top - after call
# omite el "at the bottom"
