Aclaración: Por alguna razón wordpress no me deja poner dos guiones consecutivos y los reemplaza por un único guión un poquito más largo. Cada vez que vean este guión extraño recomiendo cambiarlo por dos guiones medios para que funcione.

Hace un tiempo prepare un tutorial sobre como instalar Ruby en Ubuntu 10.04, anduvo bien, pero hoy se quedó un poco viejito. Hoy en día en la comunidad se desaconseja instalar ruby desde los repositorios de nuestras distros. Según Ryan Bigg, es: "is out-dated and leads to major headaches" [1]. La posta! es instalarlo desde RVM. Por supuesto seguimos necesitando de varias librerías y cosillas para que nuestro Ruby ande a la perfección:

sudo apt-get install build-essential git-core curl

Notese que también estamos instalando GIT

Una vez hecho esto tiramos en nuestra terminal lo siguiente:

bash < <(curl -s https://rvm.beginrescueend.com/install/rvm) echo '[[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm"' >> ~/.bashrc

Ahí mismo, RVM nos recomien instalar más cosas:

sudo apt-get install build-essential bison openssl libreadline6 libreadline6-dev curl git-core zlib1g zlib1g-dev libssl-dev libyaml-dev libsqlite3-0 libsqlite3-dev sqlite3 libxml2-dev libxslt-dev autoconf

Listo, tenemos RVM instalado perfectamente, cerramos y volvemos a abrir la terminal (para recargar el .bashrc) y vamos por Ruby:

rvm install 1.9.2 rvm install 1.8.7 # si eso no funciona: rvm package install zlib rvm remove 1.9.2 # tener cuidado con los guiones aquí abajo antes del with rvm install 1.9.2 -C --with-zlib-dir=$rvm_path/usr

Y elegimos uno para que funcione por defecto:

# cuidado con los guiones aquí también rvm --default use 1.9.2

Solo si usas versiones de Rails menores a 3: Hay un pequeño problema entre Ruby Gems y rails 2.3.x. Una de las soluciones es hacer downgrade de ruby gems: En la consola:

gem install rubygems-update -v 1.4.2 update_rubygems _1.4.2_

Y listo!

Volviendo con nuestro entorno, vamos a instalar sqlite y MySQL:

sudo apt-get install mysql-server sqlite3 sudo apt-get install libmysqlclient-dev libmysqlclient-dev

Le damos un poco de color a nuestra consola:

gem install wirble hirb

y añadimos esto a nuestro ~/.ircrb (corregido gracias a Pedro)

require 'rubygems'
require 'wirble'
require 'hirb'
Wirble.init
Wirble.colorize
# hirb (active record output format in table)
Hirb::View.enable
# IRB Options
IRB.conf[:AUTO_INDENT] = true
IRB.conf[:SAVE_HISTORY] = 1000
IRB.conf[:EVAL_HISTORY] = 200
 
# Log to STDOUT if in Rails
if ENV.include?('RAILS_ENV') && !Object.const_defined?('RAILS_DEFAULT_LOGGER')
  require 'logger'
  RAILS_DEFAULT_LOGGER = Logger.new(STDOUT)
  #IRB.conf[:USE_READLINE] = true
 
  # Display the RAILS ENV in the prompt
  # ie : [Development]>>
  IRB.conf[:PROMPT][:CUSTOM] = {
   :PROMPT_N => "[#{ENV["RAILS_ENV"].capitalize}]>> ",
   :PROMPT_I => "[#{ENV["RAILS_ENV"].capitalize}]>> ",
   :PROMPT_S => nil,
   :PROMPT_C => "?> ",
   :RETURN => "=> %s\n"
   }

  # Set default prompt
  IRB.conf[:PROMPT_MODE] = :CUSTOM
end
 
# We can also define convenient methods (credits go to thoughtbot)
# def me
#  User.find_by_email("me@gmail.com")

Por último instalamos rails y estamos listos:

gem install rails
